# CLAUDE.md

This file gives Claude Code context about the **otma** repository.

## Project Overview

**otma** is a brand repository. It stores the brand guidelines and assets that define how otma looks, feels, and communicates.

## Repository Structure

```
otma/
├── brand/
│   ├── README.md          # Brand overview and quick reference
│   ├── colors.md          # Color palette and usage rules
│   ├── typography.md      # Typefaces, scale, and rules
│   ├── voice.md           # Tone of voice and writing principles
│   └── logo/
│       └── README.md      # Logo files and usage guidelines
└── README.md              # Repo intro
```

## Key Conventions

- All brand guidelines are written in **Markdown**.
- File and folder names use **lowercase with hyphens** (e.g., `brand-guidelines.md`).
- Every section should have a clear **Usage Rules** block so contributors know what is and isn't allowed.

## When Editing Brand Files

- Keep tables aligned for readability.
- Always include a "Usage Rules" or "Do / Don't" section when defining visual or communication standards.
- Update `brand/README.md` if you add a new brand file or folder.
