The OTMA methodology

01

The Core Insight

Everything in a standard agency launch sequence is downstream of the first client. Most founders build brand, website, and offer copy before validating that anyone will buy — and end up with a polished product nobody has tested.

OTMA flips the sequence.

Standard sequence (what everyone does):
Brand → Offers → Website → Launch → Clients

MVP sequence (what actually works):
First client → Deliver Clarity Sprint → Build everything else from evidence

The brand foundation, site, and offer copy are real deliverables — but they belong in Week 3, not Week 1. The first week has one job: prove that someone will pay for a diagnosis and find it valuable.

The Edra AI Principle (Adapted)

Edra AI doesn't ask clients to document their processes. It reads what already exists — tickets, logs, messages — and surfaces the reality of how the business actually runs.

OTMA's version: OTMA doesn't ask clients to define what they want. It analyzes what they already have — existing brand, site, workflow friction — and surfaces the clearest path forward.

This shifts the framing from "tell us about yourself" to "give us access and we'll show you what's actually happening." Faster to sell, harder to confuse, immediately valuable.

02

The MVP: One Clarity Sprint

OTMA is viable when João can run this sequence end-to-end — once — with a real client paying real money:

Day 1

Access

Client shares existing materials — brand, site, workflow description, or internal tools

Days 2–3

Diagnose

OTMA maps current state, identifies the gap, finds the highest-leverage intervention

Day 3

Deliver

Written Situation Report + 60-minute walkthrough call with the client

What the Client Gets

  • Written Situation Report — current state, gap analysis, prioritized action list
  • 60-minute live walkthrough of findings (not a sales call)
  • Decision point: proceed with OTMA or take the plan and execute elsewhere
  • Real output, not a vague slide deck with generic "insights"

What the Clarity Sprint Is NOT

  • A free discovery call with a fancy name
  • A hook to sell more hours
  • A document full of recommendations that could apply to any company
  • A commitment to a full project — client decides after, not before
R$1.500 – R$2.000 Fixed price
3 business days

Low enough to reduce the barrier to start. High enough to filter for clients who take the problem seriously. Most sprints convert naturally into a core engagement — because after Day 3, the client sees the problem clearly for the first time.

03

The Minimal Shelf

This is everything OTMA needs to exist to sell and deliver the first Clarity Sprint. Nothing more.

Item What it is Build time
Clarity Sprint 1-pager One Google Doc or Notion page: what it is, what it produces, price, how to start 1 session
Outreach message 3 sentences João sends to a warm contact. Names the problem, names the offer, asks one question. 30 minutes
Intake questions 5 questions to get access to existing materials. Sent after the client says yes. 30 minutes
Situation Report template Structured doc: Current State → Gap Analysis → Priority Intervention → Path Forward. Write once, reuse forever. 1 session

What you do NOT need before the first sprint: a website, a visual identity, a logo, offer copy for all 5 service lines, agent configs, or a content calendar. Those come after — built from real delivery evidence, not assumptions.

04

Outreach Message

Three versions — adapt to the contact's actual situation. The rule: name the specific problem you've already observed, don't pitch services abstractly.

Version A — Positioning / Brand Problem

Message · WhatsApp / Email

"Tenho acompanhado o que você está construindo e tenho uma impressão clara de que o posicionamento ainda não está capturando o valor real do que você entrega. Estou testando um diagnóstico rápido — 3 dias, resultado escrito, R$1.500 — que mapeia exatamente onde está o gap e qual é a melhor intervenção. Faria sentido para você agora?"

Version B — Website / Conversion Problem

Message · WhatsApp / Email

"Olhei seu site e tenho uma hipótese de por que ele não está convertendo do jeito que deveria — mas quero confirmar antes de falar qualquer coisa. Estou testando um diagnóstico de 3 dias que entrega uma análise escrita e uma call de 60 minutos com o que encontrei, por R$1.500. Você toparia testar comigo?"

Version C — Workflow / Operational Problem

Message · WhatsApp / Email

"Sei que você está com fricção operacional [no processo X] — já ouvi você mencionar isso mais de uma vez. Estou testando um diagnóstico de 3 dias onde eu mapeio o que está acontecendo de verdade e entrego um relatório priorizado, R$1.500. Posso usar o seu caso como piloto?"

The last sentence of each version matters: it's a specific yes/no question, not an open invitation to "talk." Don't vary it.

05

Implementation Track

Three weeks. Two phases. One test.

Week 1 — Build the Shelf 4 items, ~3 hours total
  • Write Clarity Sprint 1-pager (what it is, what the client gets, R$1.500, how to start)
  • Write Situation Report template (4 sections: Current State, Gap Analysis, Priority Intervention, Path Forward)
  • Identify 3 warm contacts with an observable problem (brand, site, or workflow)
  • Write and send outreach to all 3 — adapt message to each person's actual situation
Week 2 — First Engagement First client
  • Run intake: send 5 questions, get access to existing materials
  • Day 1–2: analyze, map the gap, identify highest-leverage intervention
  • Day 3: write the Situation Report (using the template)
  • Day 3: deliver — share the doc, run the 60-min walkthrough call
  • After call: document everything João had to improvise — that list is the gap log
Week 3 — Build From Evidence Now the brand docs make sense
  • Fill brand-foundation.md using real answers surfaced during the sprint
  • Convert the delivery into the first case study (anonymized if needed)
  • Use the case study as the hero proof point for the future homepage
  • Start site-plan.md — copy written from something real, not hypothetical
  • Fix anything from the gap log before the second engagement

07

The Test

After the first Clarity Sprint is delivered, one question determines everything:

The Validation Question

"Did the client say something specific they hadn't thought of before — and did they want to keep working with OTMA?"

If yes: the model works. Build the rest.
If no: diagnose why before building anything else downstream.

If yes — the build sequence unlocks

  • Fill brand-foundation.md from delivery evidence
  • Run brand-dna → brand-os skills to derive full brand system
  • Build site-plan.md and homepage copy
  • Framer build from approved copy
  • Second Clarity Sprint (soft launch: 2 clients max before full release)

If no — the three most likely failure modes

  • Situation Report was too generic. Fix: observations must be specific to this client's materials — nothing that could appear in any competitor's report
  • Wrong client selected. Fix: the pilot client should be someone with an obvious, observable problem — not a favor, not a sympathetic ear
  • Delivery was a presentation, not a diagnosis. Fix: the report surfaces something the client didn't already know — if they already knew it, the diagnosis wasn't deep enough
