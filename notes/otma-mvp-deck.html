The OTMA methodology

The core insight

01

Most agency launches start with brand, website, and copy. That feels complete, but it skips the only thing that matters early on: whether someone will actually pay for the work.

OTMA flips the sequence. The first move is not polish. The first move is proof.

Standard sequence:
Brand → Offers → Website → Launch → Clients

MVP sequence:
First client → Clarity Sprint → Build the rest from evidence

The brand foundation, site, and offer copy are still important. They just belong after the first real diagnostic engagement, not before it.

The MVP is one Clarity Sprint

02
Day 1
Access
Client shares existing materials: brand, site, workflow description, or internal tools.
Days 2–3
Diagnose
OTMA maps current state, identifies the gap, and finds the highest-leverage intervention.
Day 3
Deliver
Written Situation Report plus a 60-minute walkthrough call with the client.

What the client gets

Deliverables
  • Written Situation Report with current state, gap analysis, and priorities
  • 60-minute live walkthrough of findings
  • Decision point: continue with OTMA or execute elsewhere

What it is not

Boundary
  • Not a free discovery call with a nicer name
  • Not a sales hook for more hours
  • Not a generic recommendation deck

The minimal shelf

03

Nothing else is required before the first sprint. Build only the pieces needed to sell and deliver one real engagement.

Item What it is Build time
Clarity Sprint 1-pager One page explaining what it is, what it produces, price, and how to start. 1 session
Outreach message Three sentences to a warm contact that name the problem, name the offer, and ask one question. 30 minutes
Intake questions Five questions that request access to existing materials after the client says yes. 30 minutes
Situation Report template Current State → Gap Analysis → Priority Intervention → Path Forward. 1 session

What you do not need yet: a website, a logo, offer copy for five service lines, agent configs, or a content calendar.

The outreach message

04

Each message should name a specific problem already visible in the contact's situation. The ask has to stay narrow and easy to answer.

Version A · Positioning
Tenho acompanhado o que você está construindo e tenho uma impressão clara de que o posicionamento ainda não está capturando o valor real do que você entrega. Estou testando um diagnóstico rápido — 3 dias, resultado escrito, R$1.500 — que mapeia exatamente onde está o gap e qual é a melhor intervenção. Faria sentido para você agora?
Version B · Website
Olhei seu site e tenho uma hipótese de por que ele não está convertendo do jeito que deveria — mas quero confirmar antes de falar qualquer coisa. Estou testando um diagnóstico de 3 dias que entrega uma análise escrita e uma call de 60 minutos com o que encontrei, por R$1.500. Você toparia testar comigo?
Version C · Workflow
Sei que você está com fricção operacional [no processo X] — já ouvi você mencionar isso mais de uma vez. Estou testando um diagnóstico de 3 dias onde eu mapeio o que está acontecendo de verdade e entrego um relatório priorizado, R$1.500. Posso usar o seu caso como piloto?

Implementation track

05

Three weeks. Two phases. One test.

Week 1 — Build the shelf

4 items · ~3 hours
  • Write the Clarity Sprint 1-pager
  • Write the Situation Report template
  • Identify 3 warm contacts with an observable problem
  • Write and send outreach to all 3

Week 2 — First engagement

First client
  • Run intake and request access to materials
  • Analyze, map the gap, and identify the intervention
  • Write and deliver the Situation Report
  • Document improvisations in a gap log

Week 3 — Build from evidence

Now it makes sense
  • Fill brand-foundation.md using real delivery evidence
  • Convert the delivery into the first case study
  • Use the case study as the homepage proof point
  • Start site-plan.md from what was actually learned

The five service lines

06

These are ready after the first Clarity Sprint validates the model. They are not the entry point.

# Offer Trigger Duration Price floor
1 Brand Foundation Sprint No clear positioning, commodity look 10 days R$4.500
2 Authority Website System Brand exists, website does not convert 15 days R$6.500
3 Workflow Optimization Audit Manual work that should be automated 5 days R$4.500
4 Automation Build System Process identified, ready to automate 10 days R$8.000
5 AI Agent Strategy & Implementation AI inside existing systems, not on top 15 days R$14.000

The funnel is simple: Clarity Sprint first, then identify the right service line, then proceed with a core engagement based on evidence.

The test

07

After the first Clarity Sprint is delivered, one question decides the next move.

The validation question
Did the client say something specific they had not thought of before, and did they want to keep working with OTMA?
If yes: the model works. Build the rest.
If no: diagnose why before building anything else downstream.

If yes

Unlock
  • Fill brand-foundation.md from delivery evidence
  • Run brand-dna and brand-os to derive the brand system
  • Build site-plan.md and homepage copy
  • Move into the second Clarity Sprint with a soft launch

If no

Diagnose
  • Situation Report was too generic
  • Wrong client selected
  • Delivery was a presentation, not a diagnosis
